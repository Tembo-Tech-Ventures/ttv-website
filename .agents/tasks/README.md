# .agents/tasks

Use this folder to plan and track complex or multi-step work before editing code.

- Create a task file per effort (e.g., `add-admin-role-checklist.md`, `2024-05-18-onboarding.md`).
- Recommended structure:
  - Goal and scope
  - Constraints/assumptions
  - Checklist or plan (update statuses as you work)
  - Planned tests/checks and when to run them
  - Decisions made and rationale
  - Links to logs in `.agents/logs` and relevant PRs/issues
- Update the task file as progress is made; keep the latest state visible.
