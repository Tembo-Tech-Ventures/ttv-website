# .agents/logs

Use this folder to capture work as you go. The goal is to leave a clear trail of what was attempted, what worked, and what failed.

- Create one log per day or task (e.g., `2024-05-18.md` or `auth-refresh-token.md`).
- For each entry, track:
  - Context/goal
  - Actions taken (with commands, files touched, decisions)
  - Tests/checks run (commands + results)
  - Outcomes (success/failure), notes, and follow-ups
- Append to existing logs rather than creating many tiny files.
- Link related task plans from `.agents/tasks` when relevant.
